## ドキュメント配置ルール

### ディレクトリ構成

```
docs/
├── README.md          — 全ドキュメントの索引（時系列+トピック別）
└── YYYY-MM-DD/        — 作業日ごとのディレクトリ
    └── <name>.md      — その日に書いたドキュメント
```

### ファイル名規約

- **ディレクトリ**: `YYYY-MM-DD`（ISO 形式、ハイフン区切り）
- **ファイル名**: kebab-case（`coreml-api-notes.md`、`ios-app-implementation.md`）。snake_case や日付プレフィックスは使わない（日付はディレクトリで表現する）

### 新規ドキュメント作成時の手順

1. その日のディレクトリ `docs/YYYY-MM-DD/` を作る（無ければ）
2. ドキュメントを `docs/YYYY-MM-DD/<name>.md` に作成
3. `docs/README.md` に1行追加（時系列セクションとトピック別セクションの両方）

### 既存ドキュメントの扱い

- 大幅な追記・修正は元の日付ディレクトリで継続編集してよい（履歴は git log で追える）
- 別日の作業として独立した内容を書く場合は新しい日付ディレクトリに別ファイルとして作る
