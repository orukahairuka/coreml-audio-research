# FFT と STFT の違い

音声前処理を理解する際に詰まったポイントをまとめる。

## 前提: DFT → FFT

- **DFT（離散フーリエ変換）**: フーリエ変換の数学的な定義。N個のデータに対して計算量は **N²**
- **FFT（高速フーリエ変換）**: DFT と同じ結果を、分割統治法で効率的に求めるアルゴリズム。計算量は **N log N**

FFT は DFT の「高速版」であり、出力は同じ。

## STFT は「速さ」ではなく「時間情報を残す」ための手法

STFT（短時間フーリエ変換）の S は **Short-Time**（短時間）であり、Speed ではない。

- FFT を音声全体に1回かけると、「どの周波数があるか」はわかるが「いつ鳴ったか」がわからない
- STFT は短い時間窓で区切り、各窓ごとに FFT をかけることで、時間と周波数の両方の情報を得る

## 計算量の比較

```
FFT 1回:   N log N

STFT:      フレーム数 × (窓サイズ log 窓サイズ)
           → トータルでは FFT 1回より多い
```

STFT の方が計算量は増える。その代わり「いつ・どの周波数が鳴ったか」がわかるようになるトレードオフ。

## まとめ

| | 何の話か |
|---|---|
| **FFT** | DFT を**速く計算する**アルゴリズム |
| **STFT** | 時間情報を残すために**窓ごとに FFT を繰り返す**手法 |

FFT と STFT はレイヤーが違う話であり、速度を比べるものではない。STFT の中で FFT が使われているという関係。

## 実装の流れ比較

### FFT

1. 音声全体を入力
2. FFT を 1回実行
3. 周波数スペクトルが 1つ得られる

### STFT

1. 音声全体を入力
2. ループ:
   a. 窓の位置を hop ずつずらして切り出す
   b. 窓関数（Hann窓）をかける
   c. FFT を 1回実行
   d. 結果を保存
3. 周波数スペクトルがフレーム数分得られる

核心の FFT 部分は同じで、STFT はその外側に「切り出し + ループ」を足しただけ。
「FFT を知っていれば STFT は書ける」と言える。追加で必要な知識は窓関数（Hann 窓）と hop の概念くらい。

## PronounSE での使われ方

`AudioFeatureExtractor.swift` の `stft()` メソッドで、ループ内で FFT（`vDSP_fft_zrip`）を毎フレーム呼び出している。

```swift
// ループで窓をずらしながら
for i in 0..<numFrames {
    let start = i * hopLength
    // 窓関数を適用して切り出し
    vDSP_vmul(frame, 1, window, 1, &windowedFrame, 1, vDSP_Length(fftSize))
    // FFT を1回実行
    vDSP_fft_zrip(fftSetup, &complexBuffer, 1, log2n, FFTDirection(FFT_FORWARD))
    // 結果を保存
}
```

```
入力音声 → プリエンファシス → STFT → メルフィルタバンク → dB正規化 → メルスペクトログラム [T, 256]
```
