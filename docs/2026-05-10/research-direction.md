# 研究方針の整理（2026-05-10）

`/grill-me` セッションを通じて、卒業研究の方向性・目的・公知化する貢献を確定させた記録。

## 研究の目標（一文）

> **「PronounSE の効果音合成を iPhone 単体で動くアプリとして実装し、その**リアルタイム性の実態**と**人間が許容するレイテンシ閾値**を実機計測と被験者実験の両面から明らかにする。これによりモバイル効果音合成における**実用設計指針**を提示する」**

## Abstract の核（一文要約）

> 「本研究は、Transformer ベース効果音合成モデル PronounSE を iPhone 実機に展開し、量子化レベル × 計算ユニットの組合せによるレイテンシと音質を 12 通りで実測すると共に、**効果音生成における人間のレイテンシ知覚閾値**を被験者実験で初めて定量化した。これにより、モバイル効果音合成の実用設計に資する具体的な設計指針を提示する」

---

## 公知（公開する貢献）にするもの ── 5 点

### 貢献 1: 実機 12 通りベンチマーク

- **何を**: iPhone 実機での 3 精度 × 4 計算ユニット = 12 通りの**レイテンシと音質の客観計測**
- **誰の役に立つ**: モバイル効果音合成を試みる**他の研究者・実装者**
- **新規性**: 効果音合成モデルでこの粒度の実機ベンチマークは前例なし。TTS / 画像系の量子化研究は多いが、**Transformer + HiFi-GAN 構成のモバイル実機データ**は手薄

### 貢献 2: 効果音生成における**レイテンシ知覚許容閾値**（★ 主張の核）

- **何を**: 被験者実験で「**何ミリ秒の遅延までユーザーは違和感なく使えるか**」を定量化
- **誰の役に立つ**: モバイル**音響アプリ全般**の設計者
- **新規性**: ★ **本研究の主張の核**。**効果音生成の文脈での**レイテンシ知覚閾値は誰も測っていない。「自分の声から生成 → 待つ → 聴く」というループに特化した知覚閾値は新規

### 貢献 3: **RangeDim + GPU メモリ問題**の発見と回避策

- **何を**: 「動的 shape × GPU 経路で `E5RT: No memory object bound to port`」現象の発見と、`fixed262` への切替による解決
- **誰の役に立つ**: CoreML で **Transformer / HiFi-GAN 系を実装する人**
- **新規性**: Apple 公式ドキュメントに記載なし、検索しても出てこない**実装上の罠**。実装報告として価値

### 貢献 4: **モバイル展開時の設計指針**

- **何を**: 「この用途ならこの設定」という**実用ガイドライン**
  - 配布アプリ用 → Int8 cpuAndNE（最小サイズ、許容速度）
  - 高音質優先 → Float16 cpuAndNE（音質と速度のバランス）
  - GPU 経路は避ける（速度悪化の構造的問題）
  - 入力長は知覚閾値内に収める
- **誰の役に立つ**: モバイル効果音合成を実装する人すべて
- **新規性**: 貢献 1〜3 を統合した実用知。論文単独では成立しないが、**Conclusion** として強い

### 貢献 5: **オープンソース実装**

- **何を**: iOS アプリと CoreML 変換スクリプト一式（GitHub 公開中）
- **誰の役に立つ**: 後続の研究者が実装をやり直さなくて済む
- **新規性**: 弱い。ただし**再現性の担保**として論文に書ける

## 評価実験のイメージ（貢献 2 を取るための設計）

### 主観実験：レイテンシ知覚閾値の測定

被験者にアプリで爆発音を生成してもらう際、**人工的に追加の遅延を挟む**：

| 条件 | 追加遅延 | 想定 |
|---|---|---|
| A | 0 ms | 合成時間そのまま（基準） |
| B | +500 ms | 軽い違和感の境界？ |
| C | +1000 ms | 明確な待たされ感？ |
| D | +2000 ms | 不快域？ |

→ 各条件で「待たされ感」「使いやすさ」を主観評価（5 段階リッカートなど）
→ **どこから不快になるか**の閾値が出る
→ 「Float16 cpuAndNE の RTF 0.84 は、知覚閾値の中で実用範囲か」を客観判断できる

### 想定被験者数・試行
- 15〜20 人
- 1 人あたり 4 条件 × 数試行
- アプリ評価モードで完結（PC 不要）

## 検討経緯のメモ（重要な気づき）

### 既存データから導かれていた事実
- iPhone 実機 12 通り計測完了済（mel L1: Float32 cpuOnly = 0、Float16/Int8 ≈ 12、Cos ≈ 0.977）
- Float16 cpuAndNE で RTF 0.84 達成
- Float32 + GPU 異常 → RangeDim 起因と判明
- 量子化で約 1.8 倍速、サイズ 1/4

### 「12 通り音出ました」だけでは研究にならない
- バッチテスト的な性格になる
- novelty を立てるには「**誰も示していない知見を、評価で示す**」構造が必要

### 端末性能差は当面扱わない
- まだ未検証
